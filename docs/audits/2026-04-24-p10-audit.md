## Architectural Notes

- StoreKit live-sync/deduping is intentionally **not** fixed in this pass. The current schema does not store a stable StoreKit identity on imported rows, so wiring `Transaction.updates` now would require guess-based matching and risks corrupting or duplicating user data.
