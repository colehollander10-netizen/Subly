## Architecture

Swift Package Manager local packages under `Packages/`:

- `SubscriptionStore` ‚Äî SwiftData models (`Trial`, `TrialAlert`), enums (`EntryType`, `EntryStatus`, `BillingCycle`). **The unified model has already shipped** ‚Äî do not rebuild it.
- `TrialEngine` ‚Äî pure logic for planning notification trigger dates from a `chargeDate`. Has test target as of P2.
- `NotificationEngine` ‚Äî actor wrapping `UNUserNotificationCenter`. Has `removePending(ids:)` and `NotificationCopy` as of P2.
- `TrialParsingCore` ‚Äî email-parsing heuristics (legacy from pre-pivot; still used for paste-to-prefill).
- `LogoService` ‚Äî brand logo fetching.
- `PhosphorSwift` ‚Äî external SPM dep for icon library.

App target at `Subly/*.swift` is a thin composition layer: views, coordinators, entry point.

## Non-obvious conventions (read before you violate them)

### Icons ‚Äî Phosphor only
Every in-app icon goes through `PhosphorSwift`. Pattern: `Ph.name.weight.color(SublyTheme.X).frame(width: N, height: N)`. Do **not** use `.foregroundStyle()` on Phosphor icons ‚Äî they use their own `.color()` modifier (it's a view extension via `ColorBlended`). Do **not** call `.resizable()` ‚Äî already applied inside Phosphor. Zero `Image(systemName:)` allowed in app-owned view code, including chevrons and tab bars (tab bar gets fixed in P9). SF Symbols only when Apple APIs render the icon themselves (share sheet, keyboard accessories).

### Design tokens ‚Äî no hardcoded values
All colors come from `SublyTheme.*` (defined in `Subly/GlassComponents.swift`). All fonts: SF Pro Rounded for display/numbers, SF Pro Text for body/labels. Default body weight is `.medium` ‚Äî `.regular` is disallowed. Never write hex colors or magic font sizes. See `DESIGN.md` ¬ß Color System and ¬ß Typography.

### Buttons
- `PrimaryButton()` ‚Äî lavender fill, **dark charcoal** text (`SublyTheme.background`). Lavender-on-white fails WCAG (2.05:1); lavender-on-dark-charcoal passes ~10.7:1. Do not "fix" this.
- `GhostButton()` ‚Äî transparent fill, lavender border + text.
- Retired: `TerminalButtonStyle`, `SecondaryTerminalButtonStyle`. Don't resurrect.

### FAB (`PrimaryAddButton`)
Solid lavender circle with Phosphor `plus.bold` in `SublyTheme.background`. No glass. No shadow. Pressed state: `scaleEffect(0.94)` + `.opacity(0.82)`. Preserve `.primaryTap` + `.primaryLongPress` haptics and the long-press gesture on any change.

### Haptics
`Haptics.play(.primaryTap | .save | .primaryLongPress | .sheetPresent | ‚Ä¶)`. Cases defined in `Subly/Haptics.swift`. Plan doubles down on "haptics everywhere meaningful" ‚Äî don't skip them on new interactions.

### SwiftData ‚Äî no unnecessary schema changes
Phase 1 deliberately kept the persisted `TrialAlert.AlertType` enum untouched. `.subscriptionDayBefore` (from `PlannedTrialAlert.Kind`) bridges to `.dayBefore` on write. If a phase requires a new persisted case, raise it explicitly ‚Äî don't add it mid-phase. No `VersionedSchema` migration plan exists; do not add one without a specific ask.

### Logo rendering
`ServiceIcon` renders brand logos over dark `glassFill`, **not** white. Transparent PNGs only; never wrap logos in a dark-opaque rounded rect (that was the COL-136 fix).

## Build commands (Subly repo root)

```bash
# Regenerate xcodeproj after adding files
xcodegen

# Build for generic iOS (no signing needed)
xcodebuild -project Subly.xcodeproj -scheme Subly \
  -destination 'generic/platform=iOS' \
  CODE_SIGNING_ALLOWED=NO build

# Build + install on booted simulator
xcodebuild -project Subly.xcodeproj -scheme Subly \
  -destination 'platform=iOS Simulator,id=<UDID>' \
  -derivedDataPath /tmp/subly-build CODE_SIGNING_ALLOWED=NO build

xcrun simctl install <UDID> /tmp/subly-build/Build/Products/Debug-iphonesimulator/Subly.app
xcrun simctl launch <UDID> com.colehollander.subly

# Package tests
cd Packages/TrialEngine && swift test
cd Packages/NotificationEngine && swift test
cd Packages/SubscriptionStore && swift test
```

### Simulator onboarding skip (for fresh installs)
The app gates Home behind onboarding. To skip during dev:
```bash
xcrun simctl terminate <UDID> com.colehollander.subly
xcrun simctl spawn <UDID> defaults write com.colehollander.subly hasCompletedOnboarding -bool YES
xcrun simctl launch <UDID> com.colehollander.subly
```
